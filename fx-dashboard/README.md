# FX Pulse

브라우저에서 바로 실행할 수 있는 환율 퀀트 시나리오 대시보드입니다.

## 포함 기능

- 기준 통화 / 대상 통화 선택
- 원하는 날짜 범위 2개를 직접 지정해서 수익률과 변동성 비교
- 3개월, 6개월, 12개월 예측값과 예측 밴드 표시
- 실측선, 장기 예측선, 예측 밴드를 함께 보여주는 SVG 차트
- 지정학 리스크, 금리 격차, 원자재 충격, 달러 유동성 스트레스 시나리오 입력
- 페이지 진입 또는 새로고침 시 최신 일별 데이터 자동 반영
- API 실패 시 최근 성공 데이터 캐시로 복구
- `SPY`, `AAPL`, `MSFT`, 미국채 `2Y`, `10Y`, 장단기 스프레드 표시
- 신뢰도 높은 시장 뉴스 필터링 표시
- 통화쌍 관련 뉴스 우선 정렬과 영향도 태그
- 뉴스 요약과 2026 주요 매크로 이벤트 캘린더

## 실행 방법

이 프로젝트는 이제 정적 파일만 여는 방식보다, `quant-pipeline` 백엔드와 함께 실행하는 방식이 기본입니다.

레포지토리 루트에서:

```bash
./setup_fx_dashboard.sh
./run_fx_dashboard.sh
```

그 다음 브라우저에서 `http://127.0.0.1:8787/dashboard/` 로 접속하면 됩니다.

다른 컴퓨터에서도 같은 방식으로 실행할 수 있고, 같은 네트워크의 다른 기기에서 접속하려면 아래처럼 실행하면 됩니다.

```bash
BIND_HOST=0.0.0.0 BIND_PORT=8787 ./run_fx_dashboard.sh
```

페이지를 열어둔 상태에서는 1시간마다 자동으로 다시 조회하고, 탭으로 돌아왔을 때도 최신 데이터를 재요청합니다.

Docker로 실행하고 싶다면 레포지토리 루트에서 아래처럼 실행할 수 있습니다.

```bash
docker build -t asterion-macro .
docker run --rm -p 8787:8787 asterion-macro
```

## 데이터 소스

- [Frankfurter API](https://frankfurter.dev/)
- [Alpha Vantage API](https://www.alphavantage.co/documentation/)

API 키 없이 일별 환율 데이터를 조회합니다.
현재 코드는 공식 `frankfurter.dev/v1` 경로를 우선 사용하고, 실패하면 구 경로를 한 번 더 시도합니다.
타임아웃, 재시도, 브라우저 `localStorage` 캐시 fallback을 포함해 네트워크 실패에 조금 더 강하게 구성했습니다.

주식 데이터는 기본적으로 Twelve Data의 demo 경로를 사용해 자동 조회합니다.
첫 구현에서는 `SPY`, `AAPL`, `MSFT`처럼 대표 자산 몇 개를 우선 사용합니다.
미국채 데이터는 미국 재무부 공식 수익률 XML 데이터를 사용해 별도 키 없이 자동 조회합니다.

뉴스는 Alpha Vantage 뉴스 엔드포인트를 사용하며, 화면에서는 `Reuters`, `AP`, `Bloomberg`, `FT`, `WSJ`, `CNBC` 같은 신뢰도 높은 도메인만 우선 노출하도록 필터링합니다.
뉴스는 선택한 통화쌍의 키워드와 관련 자산을 기준으로 우선순위를 계산해 정렬하고, `FX`, `Bond`, `Equity`, `High Impact` 같은 태그를 붙입니다.
매크로 이벤트 캘린더는 2026년 공식 일정 기준의 주요 이벤트를 노출합니다.

## 모델 참고

현재 예측은 아래 요소를 합친 프론트엔드 기반 정량 모델입니다.

- 최근 수익률 모멘텀
- 180거래일 평균 대비 평균회귀
- 최근 60거래일 연율 변동성
- 사용자가 입력한 거시 / 국제정세 시나리오 팩터

현재 국제정세 값은 자동 뉴스 수집이 아니라 사용자가 조정하는 시나리오 변수입니다.
또한 금융시장 지표(VIX, 금리, 주가지수, 원자재 등)는 아직 별도 API가 연결되어 있지 않습니다.
실제 퀀트 투자 수준으로 자동화하려면 뉴스 API, 금리/물가/원자재 데이터, 금융시장 API, 백엔드 모델링 파이프라인을 추가로 연결하는 것이 좋습니다.
